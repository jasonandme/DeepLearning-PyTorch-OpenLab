# 项目路线图

DeepLearning-PyTorch-OpenLab 的目标是保持基础内容清晰、可运行，并逐步增加更完整的实验和工程能力。

## 短期计划

短期重点是完善当前基础学习闭环，让初学者克隆项目后能顺利运行、理解和修改。

- 完善 Notebook 输出说明和实验观察。
- 为每个 Notebook 增加更明确的运行环境提示。
- 增加更多最小代码示例，覆盖常见 shape 和 dtype 问题。
- 在 `results/` 中补充轻量级实验结果示例。
- 增加统一的随机种子和绘图工具函数。
- 完善 README 中的运行截图或结果示意图。

## 中期计划

中期重点是扩展实验广度，并增强可复现性。

- 增加更多小型数据集实验，例如 Fashion-MNIST、CIFAR-10 子集和简单文本分类。
- 增加模型对比实验，例如线性模型 vs MLP、MLP vs CNN、RNN vs LSTM。
- 增加优化器和学习率对比实验。
- 增加训练日志保存功能，例如 CSV 或 JSON 格式记录 loss 和 accuracy。
- 增加 checkpoint 保存与加载示例。
- 增加简单推理 Demo，展示如何加载模型并预测单个样本。

## 长期计划

长期重点是让项目从基础教程扩展为更完整的 PyTorch 实践资源。

- 实现更完整的 Transformer Encoder 示例。
- 增加多头注意力、位置编码、残差连接和 LayerNorm 的拆解实验。
- 增加轻量部署示例，例如 TorchScript 或简单 Web API。
- 增加配置化训练入口，支持通过参数切换数据集、模型和训练设置。
- 增加测试用例，检查核心数据处理、模型输出 shape 和训练脚本可用性。
- 整理常见实验模板，方便贡献者添加新的 Notebook。

## 维护原则

- 优先保证示例可运行，再考虑扩展复杂度。
- 优先使用小型数据集或合成数据，降低运行门槛。
- 每个新增实验都应包含目标、设置、观察和可改进点。
- 文档和代码保持同步，避免只新增代码而缺少解释。
- 不把基础实验包装成算法创新，重点放在学习、复现和工程整理。
