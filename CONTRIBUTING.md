# 贡献指南

感谢你愿意改进 DeepLearning-PyTorch-OpenLab。这个项目关注基础概念、可运行实验和清晰表达，欢迎提交问题、修复和新的学习实验。

## 如何提交 issue

提交 issue 时建议说明：

- 你遇到的问题或想改进的内容。
- 操作系统、Python 版本、PyTorch 版本。
- 相关 Notebook 或脚本路径。
- 如果是报错，请提供完整错误信息和最小复现步骤。

## 如何提交新的 Notebook

新的 Notebook 建议放在 `notebooks/` 目录，文件名使用两位编号和简短英文主题，例如：

```text
07_optimizer_comparison.ipynb
```

Notebook 建议包含以下结构：

1. 本节目标
2. 背景问题
3. 核心概念
4. 公式理解
5. PyTorch 实现
6. 实验观察
7. 常见错误
8. 概念问答
9. 小结

请尽量使用小型数据集或合成数据，避免默认下载大型文件。

## 如何提交 bug fix

提交 bug fix 前建议先确认：

- 修改是否只影响相关 Notebook、脚本或文档。
- 是否保留了原有示例的可运行性。
- 是否补充了必要的说明、注释或检查项。
- 是否更新了相关实验记录或调试指南。

## Notebook 风格要求

- 面向初学者，用简洁语言解释概念。
- 每个核心概念尽量配一个小代码例子。
- 不堆叠大段理论，重点说明代码、现象和观察。
- 图表使用 Matplotlib，不使用 seaborn。
- 训练实验应设置随机种子，尽量保证结果可复现。
- 线性回归、MLP、CNN 等训练实验建议展示 loss 曲线。

## 代码风格要求

- Python 代码保持清晰、直接、可读。
- 函数和变量命名表达含义，避免过度缩写。
- 注释用于解释不明显的计算或设计选择。
- 优先使用 PyTorch、NumPy、Matplotlib 和 scikit-learn 的常见接口。
- 不提交大型数据文件、缓存文件、环境目录或中间生成文件。

## Pull Request 建议

提交 PR 时建议包含：

- 修改内容概述。
- 运行过的命令或 Notebook。
- 对实验结果的简短观察。
- 是否影响已有文档或示例。
